## Skill 定义

Skill 是**可复用的能力模块**，特点：

1. **单一职责**：每个 Skill 只做一件事
2. **明确的输入输出**：有清晰的接口定义
3. **可被调用**：由主 Agent 在适当时机调用
4. **不维护状态**：无持久化状态，纯函数式

---

## Skill 使用方式

### 方式一：主 Agent 直接调用

```
用户: "生成代码"
主 Agent: 
  → 读取技术规格书
  → 调用 java-code-generation Skill
  → 生成代码文件
```

### 方式二：通过 Sub-Agent 委托

```
用户: "委托: 生成代码"
主 Agent:
  → 创建 Sub-Agent
  → Sub-Agent 读取对应的 Skill 文件
  → 执行代码生成
  → 返回结果
```

## Skill 文件格式

```yaml
---
name: skill-name
description: Skill 描述
tools: Read, Write, Grep, Glob
---

# Skill 标题

## 触发条件

  何时使用此 Skill。

## 输入

- 输入项 1
- 输入项 2

## 输出

- 输出项 1
- 输出项 2

  ## 工作流程

  详细的工作步骤...

## 返回格式

```

状态：已完成/失败/需要决策
报告：workspace/xxx.md
产出：N 个文件
决策点：[如有]

```
```

## 新增 Skill

如需新增 Skill：

1. 创建 `{skill-name}.md` 文件
2. 遵循上述文件格式
3. 更新本 README 的 Skill 列表
4. 在 `.qoder/README.md` 中更新索引
